For on-campus SDE interviews for B.Tech freshers, internship discussions are usually used to test:

- depth of understanding,
- ownership,
- practical engineering thinking,
- communication,
- debugging ability,
- and whether you actually built what’s on your resume.

Interviewers often spend **20–60%** of the technical interview on internship/projects if you’re a fresher.

Here’s a structured “syllabus” of almost everything commonly asked from internship experience.

---

# 1. Internship Overview Questions

These are almost guaranteed.

## Typical Questions

- Tell me about your internship.
- What was your role?
- What exactly did you work on?
- What problem were you solving?
- What was the business use case?
- Why was this project needed?
- What was your biggest contribution?
- What did you learn?
- What challenges did you face?
- What would you improve if given more time?
- Which part are you most proud of?
- Explain your internship in 1 minute / 5 minutes.
- Explain it to a non-technical person.
- Explain it to a senior engineer.

---

# 2. Resume Deep Dive (VERY IMPORTANT)

Anything written can become a question.

If your resume says:

- “Improved API latency by 30%”
- “Built scalable backend”
- “Implemented authentication”
- “Used React + Node.js”
- “Worked on microservices”

…be prepared for deep follow-ups.

## Common Resume Drill Questions

- How exactly did you improve performance?
- How did you measure improvement?
- What bottleneck existed?
- What tools did you use?
- What alternatives did you consider?
- What tradeoffs were involved?
- How many users/data requests?
- How scalable was your solution?
- What happened internally when this API was called?

---

# 3. Architecture / System Design Questions (Beginner Level)

Freshers usually get simplified design discussions.

## Questions

- Explain the architecture of your project.
- Draw the system design.
- How do frontend and backend communicate?
- Why did you choose this tech stack?
- Why SQL/NoSQL?
- Why MongoDB instead of MySQL?
- Why REST instead of GraphQL?
- How are APIs structured?
- How is data stored?
- How does authentication work?
- How is deployment done?
- What happens when a user clicks this button?
- Explain end-to-end request flow.

---

# 4. Backend Questions

If your internship involved backend work.

## APIs

- What is REST?
- GET vs POST vs PUT vs PATCH?
- What status codes did you use?
- How did you validate requests?
- How did you handle errors?
- What is middleware?
- How did routing work?

## Server Concepts

- What is multithreading?
- What is async programming?
- What is concurrency?
- Difference between process and thread?
- What happens when many users hit the server simultaneously?

## Scaling

- How would you scale your backend?
- What if traffic becomes 100x?
- How would caching help?
- What is load balancing?
- What database bottlenecks can occur?

---

# 5. Database Questions

Extremely common.

## SQL

- Explain your schema.
- Primary key vs foreign key?
- What joins did you use?
- What indexes are there?
- Why indexing improves performance?
- ACID properties?
- Normalization?
- Write SQL queries based on your project.

## NoSQL

- Why NoSQL?
- Difference between SQL and MongoDB?
- Why schema-less databases?
- How is data organized?

## Optimization

- Slow query debugging?
- How would you optimize database calls?
- What if database size becomes huge?

---

# 6. Frontend Questions

If React/Angular/Vue/frontend mentioned.

## Core

- Explain component lifecycle.
- State vs props?
- What is virtual DOM?
- How does rendering happen?
- Controlled vs uncontrolled components?
- How did you manage state?

## Performance

- Why frontend became slow?
- Lazy loading?
- Memoization?
- Debouncing/throttling?
- API optimization?

# 8. DSA Questions Based on Internship

Very common pattern:
They ask implementation details from your project and convert them into DSA.

## Examples

- How did you search data efficiently?
- Which data structures were used?
- Why hashmap over array?
- Why queue here?
- Complexity of your API?
- How would your recommendation engine work?
- How would autocomplete work?
- How would caching be implemented?

# 9. Debugging & Problem Solving Questions

Interviewers love these.

## Questions

- Describe a bug you faced.
- Hardest bug?
- How did you debug it?
- How do you debug production issues?
- What tools/logs did you use?
- How would you identify memory leak?

# 16. STAR Format Questions

Behavioral rounds often expect STAR:

- Situation
- Task
- Action
- Result

Example:
“Tell me about a challenge you faced during internship.”

---

# 17. Questions That Freshers Often Fail

These eliminate many candidates.

## Dangerous Areas

- Cannot explain own code
- Used frameworks blindly
- Fake resume lines
- Cannot explain architecture
- Don’t know why a technology was chosen
- No metrics/numbers
- No debugging examples
- Shallow answers

# 20. Reality of On-Campus Freshers Interviews

Most companies do NOT expect:

- advanced distributed systems,
- production-scale expertise,
- perfect architecture.

They mainly test:

- honesty,
- clarity,
- fundamentals,
- problem solving,
- depth over buzzwords.

A fresher who deeply understands one real internship/project usually performs better than someone who memorized many technologies superficially.
