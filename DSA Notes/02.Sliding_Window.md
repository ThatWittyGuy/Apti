# Sliding Window Pattern (DSA) — Complete In-Depth Notes

The Sliding Window technique is one of the highest-frequency interview patterns. It is mainly used for **arrays**, **strings**, and **contiguous subarrays/substrings**.

---

# 1. Core Intuition

Imagine a window covering a portion of an array.

Instead of recalculating answers for every subarray from scratch, we:

- Expand the window
- Shrink the window when needed
- Reuse previously computed information

This reduces complexity from:

- Brute Force → O(n²), O(n³)
- Sliding Window → O(n)

---

## Example

Array:

```text
[2,1,5,1,3,2]
```

Find maximum sum of subarray of size 3.

Brute force:

```text
2+1+5 = 8
1+5+1 = 7
5+1+3 = 9
1+3+2 = 6
```

O(n\*k)

Sliding Window:

```text
window = [2,1,5] sum=8

remove 2
add 1

window = [1,5,1] sum=7

remove 1
add 3

window = [5,1,3] sum=9
```

Only one add and one remove each step.

O(n)

# Pattern 1: Fixed Size Window

Window size always remains K.

```cpp
while(right < n)
{
    add(nums[right]);

    if(window_size < k)
        right++;

    else if(window_size == k)
    {
        answer update;

        remove(nums[left]);
        left++;
        right++;
    }
}
```

---

## Example 1

Maximum sum subarray of size K

```cpp
int maxSum(vector<int>& nums, int k)
{
    int left = 0;
    long long sum = 0;
    long long ans = 0;

    for(int right=0; right<nums.size(); right++)
    {
        sum += nums[right];

        if(right-left+1 == k)
        {
            ans = max(ans,sum);

            sum -= nums[left];
            left++;
        }
    }

    return ans;
}
```

Complexity:

```text
O(n)
```

# Pattern 2: Variable Size Window

Window size changes dynamically.

Goal:

```text
Longest
Shortest
Maximum Length
Minimum Length
```

---

## Generic Template

```cpp
int left = 0;

for(int right=0; right<n; right++)
{
    add(nums[right]);

    while(condition invalid)
    {
        remove(nums[left]);
        left++;
    }

    ans = max(ans,right-left+1);
}
```

---

# Why It Works

Every element:

```text
Enters window once
Leaves window once
```

Total operations:

```text
2N
```

Hence:

```text
O(n)
```

# Example 2

Longest Subarray with Sum ≤ K

```cpp
int longestSubarray(vector<int>& nums, int k)
{
    int left=0;
    long long sum=0;
    int ans=0;

    for(int right=0; right<nums.size(); right++)
    {
        sum += nums[right];

        while(sum > k)
        {
            sum -= nums[left];
            left++;
        }

        ans=max(ans,right-left+1);
    }

    return ans;
}
```

# Pattern 3: Longest Substring Without Repeating Characters

Most famous interview question.

Problem:

```text
abcabcbb
```

Answer:

```text
abc
length = 3
```

---

## Intuition

Keep frequency map.

If duplicate appears:

```text
shrink window
until valid again
```

---

## Code

```cpp
int lengthOfLongestSubstring(string s)
{
    unordered_map<char,int> freq;

    int left=0;
    int ans=0;

    for(int right=0; right<s.size(); right++)
    {
        freq[s[right]]++;

        while(freq[s[right]] > 1)
        {
            freq[s[left]]--;
            left++;
        }

        ans=max(ans,right-left+1);
    }

    return ans;
}
```

Complexity:

```text
O(n)
```

# Pattern 4: At Most K

Extremely important.

Many hard problems become:

```text
At Most K
```

---

## Template

```cpp
for(right)
{
    add();

    while(violates k)
    {
        remove();
        left++;
    }

    ans += right-left+1;
}
```

---

# Why ans += right-left+1?

Every valid subarray ending at right:

```text
[left,right]
[left+1,right]
[left+2,right]
...
[right,right]
```

Count:

```text
right-left+1
```

---

# Example

Count subarrays with at most K distinct.

```cpp
long long atMostK(vector<int>& nums, int k)
{
    unordered_map<int,int> freq;

    int left=0;
    long long ans=0;

    for(int right=0; right<nums.size(); right++)
    {
        freq[nums[right]]++;

        while(freq.size() > k)
        {
            freq[nums[left]]--;

            if(freq[nums[left]]==0)
                freq.erase(nums[left]);

            left++;
        }

        ans += right-left+1;
    }

    return ans;
}
```

# Pattern 6: Minimum Window Problems

Goal:

```text
smallest window satisfying condition
```

Examples:

```text
Minimum Window Substring
Smallest Subarray with Sum >= K
```

---

## Template

```cpp
for(right)
{
    add();

    while(window valid)
    {
        answer update;

        remove();
        left++;
    }
}
```

Notice:

Longest:

```text
shrink while invalid
```

Minimum:

```text
shrink while valid
```

Very important distinction.

---

# Example: Minimum Size Subarray Sum

```cpp
int minSubArrayLen(int target, vector<int>& nums)
{
    int left=0;
    long long sum=0;
    int ans=INT_MAX;

    for(int right=0; right<nums.size(); right++)
    {
        sum += nums[right];

        while(sum >= target)
        {
            ans=min(ans,right-left+1);

            sum-=nums[left];
            left++;
        }
    }

    return ans==INT_MAX ? 0 : ans;
}
```

# Pattern 7: Frequency Map Window

Used for strings.

Maintain:

```cpp
unordered_map<char,int>
```

or

```cpp
vector<int> freq(26);
```

---

# Example: Find All Anagrams

```cpp
vector<int> findAnagrams(string s, string p)
{
    vector<int> ans;

    vector<int> need(26,0);
    vector<int> have(26,0);

    for(char c:p)
        need[c-'a']++;

    int k=p.size();

    for(int i=0;i<s.size();i++)
    {
        have[s[i]-'a']++;

        if(i>=k)
            have[s[i-k]-'a']--;

        if(have==need)
            ans.push_back(i-k+1);
    }

    return ans;
}
```

# Pattern 8: Monotonic Deque Window

For maximum/minimum in every window.

Very common.

---

## Problem

```text
Sliding Window Maximum
```

LeetCode 239.

---

## Intuition

Maintain decreasing deque.

Front always stores maximum.

---

## Code

```cpp
vector<int> maxSlidingWindow(vector<int>& nums, int k)
{
    deque<int> dq;
    vector<int> ans;

    for(int i=0;i<nums.size();i++)
    {
        while(!dq.empty() && dq.front()<=i-k)
            dq.pop_front();

        while(!dq.empty() &&
              nums[dq.back()]<=nums[i])
            dq.pop_back();

        dq.push_back(i);

        if(i>=k-1)
            ans.push_back(nums[dq.front()]);
    }

    return ans;
}
```

Complexity:

```text
O(n)
```

# Final Mental Model

Think of sliding window as **"grow → validate → shrink → record answer"**.

There are only 5 interview-grade subpatterns you need to master:

1. **Fixed Size Window**
2. **Longest Valid Window**
3. **Minimum Valid Window**
4. **Count Using AtMost**
5. **Monotonic Deque Window**

If you can identify which of these five templates a problem belongs to, you can solve roughly **80–90% of sliding-window interview questions** in O(n) time.
