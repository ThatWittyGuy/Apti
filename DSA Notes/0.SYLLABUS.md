# Golden Interview Rule

Before solving any problem ask:

1. Can I use a HashMap?
2. Can I sort?
3. Can I use Two Pointers?
4. Can I use Sliding Window?
5. Can I use Binary Search?
6. Is this a Graph?
7. Is this DP?

Most interview problems fall into one of these patterns.

For interview preparation, the highest ROI order is:

1. Arrays
2. Hashing
3. Two Pointers
4. Sliding Window
5. Binary Search
6. Stack / Monotonic Stack
7. Heap
8. Trees
9. Graphs
10. Dynamic Programming

Mastering those covers the majority of coding interview questions asked at top product companies.
